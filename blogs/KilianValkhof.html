Weekly Nerd 2024 — Sessie #01

Kilian
Valkhof

Frontend development, CSS en het eenvoudiger maken van websites. Houd het simpel.

Tijdens deze Weekly Nerd gaf Kilian Valkhof een presentatie over frontend development, CSS en het eenvoudiger maken van websites. De belangrijkste boodschap van de presentatie was dat developers hun oplossingen zo simpel mogelijk moeten houden.

Houd het simpel

Volgens Kilian moet je development niet moeilijker maken dan nodig is.

Wanneer iets met CSS opgelost kan worden, hoef je niet direct JavaScript te gebruiken. CSS-oplossingen zijn vaak sneller, overzichtelijker en beter voor performance.

Een voorbeeld hiervan is:

appearance: none;

Hiermee kun je standaard browserstijlen van formulieren en checkboxes verwijderen zodat je zelf een eigen design kunt maken.

Waarom custom form styles bestaan

Vroeger zagen checkboxes, buttons en formulieren er in iedere browser bijna hetzelfde uit. Developers wilden meer controle over het design van hun interfaces en begonnen daarom eigen stijlen te maken voor formulieren.

Met CSS kun je tegenwoordig veel meer aanpassen zonder ingewikkelde JavaScript-oplossingen.

Focus states en accessibility

Een ander belangrijk onderwerp was accessibility bij formulieren.

Standaard heeft een inputveld vaak een opvallende focusstijl wanneer iemand met een toetsenbord navigeert. Veel developers verwijderen deze stijl met: 

outline: none;

Maar dit is slecht voor accessibility, omdat toetsenbordgebruikers dan niet meer kunnen zien welk element geselecteerd is.

Daarom werd uitgelegd dat je beter een eigen focusstijl kunt ontwerpen in plaats van de outline volledig te verwijderen.

Bijvoorbeeld:

outline-color: transparent;

Dit helpt mensen die websites gebruiken met een toetsenbord in plaats van alleen een muis of touchscreen.

Smooth scrolling met CSS

Ook het gebruik van smooth scrolling werd besproken.

scroll-behavior: smooth;

kan de browser zelf scrollanimaties regelen zonder dat je hiervoor JavaScript hoeft te gebruiken.

Het voordeel hiervan is dat de browser automatisch rekening houdt met:

  • De kracht van het apparaat
  • Performance
  • Browserinstellingen
  • Accessibility voorkeuren van gebruikers

Daardoor kunnen animaties bijvoorbeeld langzamer afgespeeld worden op minder krachtige apparaten.

Normaal zouden developers dit allemaal handmatig moeten programmeren in JavaScript, maar CSS neemt dit werk grotendeels over.

Gebruikers controle geven

Een belangrijk punt uit de presentatie was dat gebruikers controle moeten houden over animaties.

Sommige mensen zetten animaties bewust uit vanwege accessibility-redenen, bijvoorbeeld omdat ze gevoelig zijn voor bewegende beelden.

Door moderne CSS-oplossingen te gebruiken kan de browser rekening houden met deze voorkeuren en een betere gebruikerservaring bieden voor iedereen.
