Weekly Nerd 2024 — Sessie #02

Peter-Paul
Koch

Browsers, accessibility en rendering engines. Denk eerst goed na over je layout.

Tijdens de Weekly Nerd gaf Peter-Paul Koch een presentatie over browsers, accessibility en hoe rendering engines werken. De talk ging vooral over hoe browsers HTML en CSS verwerken, waarom accessibility belangrijk is en hoe developers beter kunnen nadenken over hun layouts en structuur.

JavaScript als standaard engine

Volgens Peter-Paul Koch wordt JavaScript tegenwoordig gebruikt als standaard engine binnen browsers. JavaScript speelt een grote rol in interactieve websites en werkt samen met andere onderdelen van de browser, zoals de DOM en rendering engine.

Wat is AOM?

De AOM (Accessibility Object Model) lijkt veel op de DOM, maar heeft een andere focus. Waar de DOM vooral de structuur van een webpagina beschrijft, is de AOM meer gericht op accessibility.

De AOM helpt bijvoorbeeld mensen die gebruikmaken van screenreaders of andere accessibility-tools. Hierdoor kunnen websites beter toegankelijk worden gemaakt voor gebruikers met beperkingen.

Wat is een browser?

Een browser bestaat uit verschillende onderdelen die samenwerken:

  • HTML
  • CSS
  • DOM
  • AOM
  • Interface

Al deze onderdelen zorgen ervoor dat een website goed geladen, weergegeven en gebruikt kan worden.

Rendering engine

De rendering engine is verantwoordelijk voor meerdere belangrijke processen binnen een browser. Deze engine zorgt voor:

  • HTML parsing
  • CSS parsing
  • Rendering van de pagina
  • Constructie van de DOM tree
  • Constructie van de AOM tree

De rendering engine is dus niet verantwoordelijk voor de interface of de JavaScript engine, maar vooral voor het verwerken en weergeven van content op een webpagina.

Denk eerst goed na over je layout

Een belangrijk punt uit de presentatie was dat je eerst goed moet nadenken over je layout voordat je begint met bouwen.

Wanneer je tijdens het ontwikkelen steeds grote veranderingen maakt aan de structuur of layout, kan het zogenaamde "popcorn effect" ontstaan. Hierbij blijven er steeds nieuwe problemen verschijnen doordat onderdelen van de website elkaar beïnvloeden.

Daarom is het slimmer om eerst een sterke basisstructuur te ontwerpen en daarna pas verder te bouwen.

Backward compatibility

Browsers ondersteunen vaak nog steeds oude technieken en functies. Dit heet backward compatibility.

Volgens Peter-Paul Koch betekent dit eigenlijk:

Alles wat een browser ooit heeft ondersteund, moet die browser voor altijd blijven ondersteunen.

Hierdoor blijven oudere websites vaak nog werken in moderne browsers.
