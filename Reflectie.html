Sprint 0

Tijdens deze sprint heb ik veel geleerd over front-end development, vooral op het gebied van CSS, JavaScript en interactieve websites. Mijn leerdoelen waren om beter te worden in animaties, JavaScript en samenwerken binnen een developmentomgeving.

Ik heb geleerd hoe ik data kan ophalen met een API en deze dynamisch kan tonen op mijn website met JavaScript. Daarnaast heb ik gewerkt met interactieve SVG's, waarbij gebruikers op districten kunnen klikken om informatie te bekijken. Dit hielp mij beter begrijpen hoe HTML, CSS en JavaScript samenwerken.

Ook heb ik nieuwe CSS technieken toegepast, zoals masking, dark & light mode, animaties en een custom cursor. Hierdoor kreeg mijn website een professionelere uitstraling en leerde ik meer over gebruikerservaring en vormgeving.

Soms werkte niet alles direct, zoals hover effecten of het gebruiken van een video in een SVG. Door hiermee te experimenteren heb ik geleerd beter problemen op te lossen en kritischer naar mijn code te kijken tijdens code reviews.

Ik merk dat ik meer controle krijg over het combineren van design en code. Vooral met CSS-animaties, API's en interactieve elementen heb ik veel vooruitgang geboekt.

Browser Tech

Wat heel goed ging is dat ik makkelijk de nieuwe code kon snappen over de fieldset en labels en inputs. In het begin moest ik de hiërarchie goed onder de knie krijgen, maar dat ging snel. Ik vind dat dit goed ging want ik had niet veel tijd nodig om het toe te passen.

Wat minder goed ging is het gebruik van JavaScript voor de validatie. Het werd alleen toegepast op de eerste input. Dus heb ik een alternatief gebruikt met display: block en display: none.

Waarmee ik meer vast zat was de progressive disclosure laten werken met CSS:

.divVan1b {
  &:has(input[name="1bEersteVraag"][value="nee"]:checked) + .divVan1c ._1c-1,
  &:has(input[name="1bTweedeVraag"][value="nee"]:checked) + .divVan1c ._1c-1,
  &:has(input[name="1bDerdeVraag"][value="nee"]:checked) + .divVan1c ._1c-1,
  &:has(input[name="1bDerdeVraag"][value="ja"]:checked) + .divVan1c,
  &:has(input[name="1bDerdeVraag"][value="ja"]:checked) + .divVan1c ._1c-1 {
    display: block;
  }
}

Wat ik anders zou doen volgende keer is meer tijd besteden om JavaScript beter te leren.

Over mijn leerdoelen

Ik wilde meer leren werken met JavaScript, maar ik vind dat ik het niet goed genoeg heb gehaald want ik heb weinig gedaan met JavaScript. Wat me het meest heeft verrast is dat HTML veel meer dingen heeft dan ik dacht en dat ik het heb kunnen leren en toepassen.

Over mijn code

Ik ben het meest trots op de algemene styling van mijn website en de progressive disclosure die ik met alleen CSS werkend heb gekregen. Wat ik nog niet goed genoeg vind is dat de knoppen niet naast elkaar zijn op een groter scherm, en de JavaScript code die zeer nauw is.

CSS To The Rescue

Over het proces

Wat goed ging is dat ik de controller makkelijk kon stylen. Maar ook het toepassen van nieuwe technieken zoals nesting in CSS en container queries voor de thema's. Aan het begin was het lastig maar nu weet ik het beter toe te passen.

Wat minder goed ging was het gebruiken van @property. Ik wou een balletje laten bewegen maar het lukte niet. Nu wel, want ik heb geleerd hoe ik het kan gebruiken:

@property --x {
  syntax: "";
  inherits: true;
  initial-value: 0.5;
}

@property --y {
  syntax: "";
  inherits: true;
  initial-value: 0.5;
}

transition: --x 10000s linear, --y 10000s linear;

&:has(input[value="omhoog"]:checked) { --y: 0; }
&:has(input[value="omlaag"]:checked)  { --y: 1; }
&:has(input[value="rechts"]:checked)  { --x: 1; }
&:has(input[value="links"]:checked)   { --x: 0; }

&:has(input[value="omhoog"]:checked),
&:has(input[value="omlaag"]:checked) {
  transition: --x 10000s linear, --y 1.5s linear;
}

&:has(input[value="rechts"]:checked),
&:has(input[value="links"]:checked) {
  transition: --x 1.5s linear, --y 10000s linear;
}

Wat ik volgende keer beter zou doen is zoveel mogelijk gebruik maken van nieuwe code, zodat ik het sneller en beter leer.

Over mijn leerdoelen

Ik wou meer leren over JavaScript en CSS animaties. Het stukje over CSS animaties heb ik beter geleerd. Niet 100% maar wel beter dan daarvoor. Ik weet wat ik moet gebruiken om iets te animeren en hoe.

Over de code

Ik ben het meest trots op de code over het balletje — het beweegt met knoppen zonder JavaScript. Ook vind ik het crazy thema heel leuk en hoe alles verandert met de container queries. Wat ik minder goed vind is de structuur en de comments. Meer commentaar zou de code leesbaarder maken.

Project Nebula Explorer

Ik vond dit een erg leuk en uitdagend project om aan te werken. Het ruimte-thema gaf ons veel creatieve vrijheid, waardoor we echt konden brainstormen over vette features en easter eggs. De samenwerking binnen het team verliep heel soepel en snel; we konden makkelijk schakelen tussen het ontwerp in Figma en de uiteindelijke code. Ik heb in korte tijd ontzettend veel geleerd en ik ben erg trots op het eindresultaat dat we als groep hebben neergezet.

CSS & Animaties (UX): Mijn doel was om meer te leren over CSS en hoe je animaties functioneel toepast. Door te werken met scroll-driven animations en een custom cursor heb ik hier veel ervaring mee opgedaan. Een belangrijk leermoment was het toevoegen van reduced-motion. Hierdoor heb ik geleerd hoe je een website niet alleen visueel aantrekkelijk, maar ook toegankelijk maakt voor iedere gebruiker.

JavaScript-verdieping: Ik wilde mijn JavaScript-kennis uitbreiden naar een hoger niveau. Voor de 3D-satelliet moest ik echt verder kijken dan de basis. Ik heb JavaScript gebruikt om specifieke coördinaten (data-position) uit het 3D-model te halen voor de interactieve popovers. Ook het aansturen van de geluidseffecten op de juiste momenten was een goede oefening.

Samenwerken & Git: Met het oog op mijn aankomende front-end stage wilde ik beter leren samenwerken in een code-omgeving. Toen we op maandag per ongeluk de developer-branch verwijderden, was dat even stressvol, maar achteraf juist heel leerzaam. Het dwong ons om meteen strakke afspraken te maken over Git-beheer en communicatie. Dit is een waardevolle ervaring die ik zeker meeneem naar mijn stage.

API

Mijn doel van dit project was om een werkende website te maken en nieuwe dingen te leren, vooral dingen met API. Ik zou zeggen dat dit een succes is. Want ik heb een goed werkend product gemaakt.

De onderdelen die het best werken zijn het ophalen van de data van de API en deze tonen op de website. Het gedeelte van latitude en longitude ophalen en tonen werkt beter dan ik had verwacht, maar uiteindelijk was het toch simpeler.

Wat ik het moeilijkst vond was in het begin alles opnieuw leren. Qua code vond ik het lastigste gedeelte het stukje waarbij ik de data ophaal voor de sidebar. Ik heb het eerst opgehaald, in een array gezet en die array gereturned — een grote omweg. Bij de detailspagina heb ik het daarna beter gedaan.

In het kort wat ik heb geleerd: hoe ik met Astro moet werken, if/else anders schrijven met JSX en ternary operators, API data ophalen en tonen, werken met Leaflet, Ticketmaster, Geolocation en LocalStorage, en meer vragen stellen aan docenten.

Wat beter ging dan verwacht is hoe snel ik de code snapte. En wat ik verder wil leren is werken met andere frameworks zoals Three.js. Dit project heeft me veel geholpen groeien als ontwikkelaar.

Human Centred Design

Tijdens dit project heb ik gewerkt met de Exclusive Design principles: study situation, ignore conventions, prioritise identity en add nonsense. Deze kwamen terug bij het ontwerpen van een accessibility extensie voor Ihab, die blind is.

Study situation

Ik heb gekeken naar de situatie van Ihab en hoe hij websites gebruikt met een screenreader en toetsenbord. Hieruit volgden functies als sneltoetsen (Alt + 1 t/m 4), focus management en opgeslagen posities. Ik heb zelf ook NVDA getest om beter te begrijpen hoe een screenreader werkt.

Ignore conventions

Dit zie je terug in keuzes zoals focusbare headings en Alt + S om een positie op te slaan. Geen standaardinteracties, maar voor Ihab juist beter.

Prioritise identity

Ik ontwierp niet voor een standaardgebruiker maar specifiek voor iemand die blind is. Tijdens user tests luisterde ik naar de feedback van Ihab — zo wilde hij audio in plaats van popup meldingen, en sommige sneltoetsen botsten met die van zijn screenreader.

Add nonsense

Dit zie je terug in kleine details, zoals een anime punch-geluid bij het verwijderen van een opgeslagen positie. Ihab vond dit grappig en het maakt de extensie persoonlijker.

Door deze principles toe te passen heb ik geleerd dat je niet altijd standaard regels hoeft te volgen, en hoe belangrijk testen en itereren is binnen Human Centred Design.

Meestersproef

Tijdens dit project heb ik gewerkt aan verschillende onderdelen van de applicatie, zoals de header, de zoekfunctie met Algolia, de list-view en het responsive maken van de website. Daarnaast heb ik geholpen met het verbeteren van de toegankelijkheid volgens de WCAG-richtlijnen.

Door de zoekfunctie te bouwen heb ik geleerd hoe ik een externe dienst zoals Algolia kan koppelen aan een website. Ook heb ik geleerd hoe een debounce werkt. Hierdoor wordt er niet bij elke toetsaanslag direct een zoekopdracht verstuurd, wat zorgt voor betere prestaties van de applicatie. Daarnaast heb ik meer ervaring opgedaan met het werken met API's, fetch-requests en environment variables.

Bij de list-view heb ik geleerd hoe verschillende componenten met elkaar kunnen communiceren via Custom Events. Hierdoor begreep ik beter hoe data tussen verschillende onderdelen van een applicatie kan worden uitgewisseld.

Ook heb ik veel geleerd over responsive design en toegankelijkheid. Omdat de website bedoeld is voor mensen met een lichamelijke beperking, moesten we goed nadenken over zaken zoals toetsenbordnavigatie, screenreaders en ARIA-attributen. Hierdoor ben ik mij meer bewust geworden van het belang van toegankelijk ontwerpen.

Over mijn leerdoelen

Mijn persoonlijke leerdoel was beter leren samenwerken binnen een team. Tijdens dit project heb ik geleerd om vaker te overleggen, feedback te geven en ontvangen en mijn werkzaamheden goed af te stemmen met mijn groepsleden. Hierdoor verliep de samenwerking beter en konden we samen een sterker eindproduct maken.

Mijn leerdoel was ook om meer javascript te leren schrijven, dit heb ik gedaan door middel van het bouwen van de zoekfunctie en de list-view. Tijdens de zoekfuncite heb ik nieuwe dingen geleerd, zoals costum events en debounce.

Over de code

Ik ben het meest trots op de zoekfunctie met Algolia, de list-view en de responsive verbeteringen. Deze onderdelen hebben veel waarde toegevoegd aan het eindproduct en zorgen ervoor dat de website toegankelijker en gebruiksvriendelijker is. Daarnaast heb ik geleerd hoe belangrijk feedback van gebruikers, groepsleden en opdrachtgevers is om een product steeds verder te verbeteren.
